Сборка и установка udpxy с помощью gcc или Clang компилятора:

Разархивировать исходный код из файла *.tgz в директорию по выбору с помощью команды:

tar -xzvf udpxy.X.Y-ZZ.tgz or gzip -dc udpxy.X.Y-ZZ.tgz | tar -xvf -

Убедитесь в наличии GNU make (gmake в BSD-системах).

Если нужный компилятор не совпадает с cc, используемым по умолчанию в системе, добавьте префикс к make в виде CC=your_compiler. К примеру, если clang не является C компилятором по умолчанию, но следует использовать именно его, выполните:

CC=clang make режим

Для make доступны режимы, комбинирующие результирующий размер модулей и опции отладки:

Режим:есть asserts:есть debug символы:есть режим verbose:
debugYESYESYES
releaseNONOYES
leanNONONO
rdebugNOYESYES
ldebugNOYESNO

Запуск make без указания режима выберет режим release для сборки udpxy.

После успешного окончания работы make, исполняемые модули udpxy могут быть вручную скопированы в выбранную Вами диркторию и в дальнейшем запускаться оттуда - никаких дополнительных шагов не требуется.

Можно также выполнить команду make install для инсталяции исполняемых модулей udpxy в директорию /usr/local/bin.
